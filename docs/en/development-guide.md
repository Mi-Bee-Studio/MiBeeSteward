# MiBee Steward Development Guide

This guide provides comprehensive instructions for contributing to the MiBee Steward project, covering development setup, project structure, common development tasks, and coding conventions.

## Development Setup

### Starting Development Environment

The `make dev` command starts both frontend and backend with hot reload:

```bash
# Install dependencies first
cd web && npm install
cd ..

# Start development server (frontend + backend)
make dev
```

This runs:
- Frontend: `npm run dev` on port 5173
- Backend: `go run cmd/server/main.go` on port 8080
- Hot reload enabled for both frontend and backend

### Building for Production

```bash
# Build both frontend and backend
make build

# Cross-compile for multiple platforms
make build-all              # linux amd64 + arm64
make build-linux-arm64      # arm64 only
```

## Project Structure

```
.
├── cmd/server/           # Entry point — main.go
├── configs/              # YAML config (koanf)
├── db/                   # SQL migrations + sqlc queries → internal/db/
├── internal/
│   ├── api/             # HTTP layer (Chi): handlers, middleware, routes
│   ├── config/          # Config loading (koanf/v2)
│   ├── db/              # ⚡ GENERATED by sqlc — do not edit
│   ├── domain/          # Domain models (device, user, document)
│   ├── repository/      # Data access (wraps sqlc generated code)
│   └── service/         # Business logic + probe subsystem
├── web/                 # SvelteKit 5 SPA → embedded via go:embed
├── deploy/              # Systemd service, nginx, Docker configs
├── scripts/              # backup.sh (SQLite backup with integrity check)
└── tests/integration/
```

### Key Architecture Layers

- **Domain Layer** (`internal/domain/`): DTOs, constants, request/response models
- **Repository Layer** (`internal/repository/`): Data access (optional wrapper around sqlc)
- **Service Layer** (`internal/service/`): Business logic, probe subsystem
- **Handler Layer** (`internal/api/handler/`): HTTP request handling, response formatting

## How to Add an API Endpoint

Follow these steps to add a new API endpoint:

### 1. Define Request/Response DTOs

Create or update domain models in `internal/domain/`:

```go
// internal/domain/your_domain.go
type CreateYourResourceRequest struct {
    Name        string `json:"name" validate:"required"`
    Description string `json:"description" validate:"max:255"`
}

type UpdateYourResourceRequest struct {
    Name        *string `json:"name"`
    Description *string `json:"description"`
}

type YourResourceResponse struct {
    ID          int64     `json:"id"`
    Name        string    `json:"name"`
    Description string    `json:"description"`
    CreatedAt   time.Time `json:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"`
}
```

### 2. Create sqlc Query

Add SQL queries in `db/queries/your_resource.sql`:

```sql
-- db/queries/your_resource.sql
-- name: CreateYourResource :one
INSERT INTO your_resources (name, description) 
VALUES ($1, $2) 
RETURNING *;

-- name: GetYourResource :one
SELECT * FROM your_resources WHERE id = $1;

-- name: ListYourResources :many
SELECT * FROM your_resources ORDER BY created_at DESC;
```

### 3. Generate sqlc Code

Run sqlc to generate Go code:

```bash
~/go/bin/sqlc generate
```

Generated code appears in `internal/db/`.

### 4. Add Service Method

Create service method in `internal/service/your_service.go`:

```go
// internal/service/your_service.go
type YourService struct {
    db *sqlc.Queries
}

func NewYourService(db *sqlc.Queries) *YourService {
    return &YourService{db: db}
}

func (s *YourService) CreateYourResource(ctx context.Context, req domain.CreateYourResourceRequest) (*domain.YourResourceResponse, error) {
    dbRes, err := s.db.CreateYourResource(ctx, req.Name, req.Description)
    if err != nil {
        return nil, fmt.Errorf("failed to create resource: %w", err)
    }
    return s.toYourResourceResponse(dbRes), nil
}
```

### 5. Create Handler

Create handler in `internal/api/handler/your_handler.go`:

```go
// internal/api/handler/your_handler.go
type YourHandler struct {
    service *service.YourService
}

func NewYourHandler(service *service.YourService) *YourHandler {
    return &YourHandler{service: service}
}

func (h *YourHandler) CreateYourResource(w http.ResponseWriter, r *http.Request) {
    var req domain.CreateYourResourceRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        response.Error(w, http.StatusBadRequest, "Invalid request body")
        return
    }

    res, err := h.service.CreateYourResource(r.Context(), req)
    if err != nil {
        response.Error(w, http.StatusInternalServerError, err.Error())
        return
    }

    response.Created(w, res)
}
```

### 6. Register Route

Add route in `internal/api/routes/routes.go`:

```go
// internal/api/routes/routes.go
func (r *Router) setupRoutes() {
    // ... existing routes
    
    yourHandler := handler.NewYourHandler(service.NewYourService(r.db))
    
    r.router.Group(func(r chi.Router) {
        r.Use(middleware.RequireAuth)
        r.Post("/your-resources", yourHandler.CreateYourResource)
        r.Get("/your-resources", yourHandler.ListYourResources)
    })
}
```

## How to Add a Database Query

### sqlc Workflow

1. **Write query** in `db/queries/*.sql`:
```sql
-- db/queries/your_table.sql
-- name: GetYourData :one
SELECT * FROM your_table WHERE id = $1;
```

2. **Run sqlc generate**:
```bash
~/go/bin/sqlc generate
```

3. **Use generated code** in services:
```go
// internal/service/your_service.go
func (s *YourService) GetYourData(ctx context.Context, id int64) (*db.YourData, error) {
    return s.db.GetYourData(ctx, id)
}
```

### Database Schema Changes

```sql
-- db/schema.sql
CREATE TABLE your_table (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

1. **Edit schema.sql** in `db/`:
2. **Regenerate sqlc code**:
```sql
-- db/migrations/000002_add_your_table.up.sql
CREATE TABLE your_table (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- db/migrations/000002_add_your_table.down.sql
DROP TABLE your_table;
```

```bash
~/go/bin/sqlc generate
```


## How to Add a Frontend Page

### 1. Create Route File

Create SvelteKit route with file-based routing:

```svelte
<!-- web/src/routes/your-feature/+page.svelte -->
<script lang="ts">
    import { api } from '$lib/api/client';
    import { onMount } from 'svelte';
    
    let items = [];
    let loading = false;
    let error = null;
    
    onMount(async () => {
        loading = true;
        try {
            items = await api.get('/your-resources')();
        } catch (err) {
            error = err.message;
        } finally {
            loading = false;
        }
    });
</script>

<h1>Your Feature</h1>

{#if loading}
    <p>Loading...</p>
{:else if error}
    <p class="error">{error}</p>
{:else}
    <ul>
        {#each items as item (item.id)}
            <li>{item.name}</li>
        {/each}
    </ul>
{/if}
```

### 2. Use API Client

Use the wrapped API client from `$lib/api/client.ts`:

```typescript
// Generic API calls with auto-401 handling
const data = await api.get<T>(path)();
const created = await api.post<T>(path)(data);
const updated = await api.put<T>(path)(data);
const deleted = await api.delete(path)();
```

### 3. Add i18n Translations

Update translation files:

```json
// web/messages/en.json
{
    "yourFeature": {

        "error": "Failed to load data"

}
```


## How to Add a Probe Type

### 1. Create Probe Implementation

Implement the `Prober` interface in `internal/service/probe/your_probe.go`:

```go
// internal/service/probe/your_probe.go
type YourProbe struct {
    // Add probe-specific config
}

func (p *YourProbe) Probe(ctx context.Context, target string, timeout time.Duration) (*ProbeResult, error) {
    ctx, cancel := context.WithTimeout(ctx, timeout)
    defer cancel()
    
    // Implement your probing logic
    // Return ProbeResult with status, duration, error
    return &ProbeResult{
        Status:   ProbeStatusSuccess,
        Duration: time.Since(start),
        Output:   "Probe output",
    }, nil
}
```

### 2. Register in Heartbeat Service

Update the `GetProber()` switch in `internal/service/heartbeat.go`:

```go
// internal/service/heartbeat.go
func (h *HeartbeatService) GetProber(method string) (Prober, error) {
    switch method {
    case "icmp":
        return NewICMPProber(h.config.ICMPTimeout), nil
    case "tcp":
        return NewTCPProber(h.config.TCPTimeout), nil
    case "http":
        return NewHTTPProber(h.config.HTTPTimeout), nil
    case "snmp":
        return NewSNMPProber(h.config.SNMPCommunity), nil
    case "your-probe":
        return &YourProbe{}, nil
    default:
        return nil, fmt.Errorf("unknown probe method: %s", method)
    }
}
```

### 3. Update Configuration

Add probe type to configuration:

```yaml
# configs/config.example.yaml
heartbeat:
  probes:
    - name: "your-probe"
      method: "your-probe"
      target: "your-target.example.com"
      interval: 30
      timeout: 10
```

## How to Add a Scanner Protocol (v2 engine)

The v2 scanner is plugin-based: adding a new protocol (e.g. MQTT detection) is a **classifier + handler** pair, registered at startup. No orchestrator or persistence changes needed.

### 1. Add a ProbeSource (if the protocol needs active probing)

If the new protocol isn't already discoverable from the existing TCP/banner probes, add a `ProbeSource` in `internal/service/scannerv2/probe/`:

```go
// internal/service/scannerv2/probe/mqtt.go
type MQTTProbe struct{}
func (p *MQTTProbe) Name() string { return "active:mqtt" }
func (p *MQTTProbe) Probe(ctx context.Context, ip string, hint scannerv2.ProbeHint) ([]scannerv2.Evidence, error) {
    // connect to port 1883, send CONNECT packet, emit Evidence{Kind:"mqtt_banner",...}
}
```
Register it in `probe.DefaultProbeSources()`. If the protocol is identifiable from existing banner evidence alone, skip this step.

### 2. Add a ServiceClassifier

Create a pure classifier in `internal/service/scannerv2/classify/mqtt.go` that emits a `ServiceIdentity` when it sees matching evidence:

```go
type MQTTClassifier struct{}
func (MQTTClassifier) Service() string { return "mqtt" }
func (MQTTClassifier) Classify(ev []scannerv2.Evidence) []scannerv2.ServiceIdentity {
    // scan ev for kind=="banner" with MQTT magic bytes → emit identity
}
```
Register in `classify.DefaultClassifiers()`.

### 3. Add a ServiceHandler (optional, for heartbeat/enrich)

If the service needs a custom heartbeat or device enrichment, add a handler in `internal/service/scannerv2/handler/mqtt.go` implementing `GenerateHeartbeat`, `Collect` (may return `Trigger`s for cascades), and `EnrichDevice`. Register in `handler.DefaultHandlers()`. If none of those are needed (detection alone suffices), skip this step.

### 4. Test

Unit tests live next to the source (`*_test.go`), using `net.Listen` / `httptest` for probe mocks and pure-function tests for classifiers. Run `go test ./internal/service/scannerv2/...`.

## Testing Conventions

### Test Setup

Use `testify/require` assertions and in-memory SQLite:

```go
// tests/integration/your_test.go
package integration

import (
    "testing"
    "github.com/stretchr/testify/require"
    
    "github.com/Mi-Bee-Studio/MiBeeSteward/internal/service"
)

func TestYourService_CreateYourResource(t *testing.T) {
    // Setup in-memory SQLite
    db, err := sql.Open("sqlite", ":memory:")
    require.NoError(t, err)
    defer db.Close()
    
    // Run migrations
    // ...
    
    // Create service
    svc := service.NewYourService(db)
    
    // Test
    req := domain.CreateYourResourceRequest{
        Name:        "Test Resource",
        Description: "Test description",
    }
    
    res, err := svc.CreateYourResource(context.Background(), req)
    require.NoError(t, err)
    require.NotZero(t, res.ID)
}
```

### Integration Tests with httptest.Server

```go
func TestYourHandler_CreateYourResource(t *testing.T) {
    // Setup test server
    server := httptest.NewServer(router.Setup())
    defer server.Close()
    
    // Test client
    client := server.Client()
    
    // Test data
    req := domain.CreateYourResourceRequest{
        Name:        "Test Resource",
        Description: "Test description",
    }
    
    // Make HTTP request
    resp, err := client.Post(server.URL+"/your-resources", "application/json", 
        bytes.NewReader(jsonEncode(req)))
    require.NoError(t, err)
    defer resp.Body.Close()
    
    // Verify response
    require.Equal(t, http.StatusCreated, resp.StatusCode)
    
    var res domain.YourResourceResponse
    require.NoError(t, jsonDecode(resp.Body, &res))
    require.NotZero(t, res.ID)
}
```

### Test Helpers

Use `t.Helper()` for test helpers:

```go
func requireJSONResponse(t *testing.T, resp *http.Response, expectedStatus int) {
    t.Helper()
    require.Equal(t, expectedStatus, resp.StatusCode)
    
    if expectedStatus >= 200 && expectedStatus < 300 {
        var body map[string]interface{}
        require.NoError(t, jsonDecode(resp.Body, &body))
    } else {
        var errResp map[string]string
        require.NoError(t, jsonDecode(resp.Body, &errResp))
        require.Contains(t, errResp, "error")
    }
}
```

## Code Style & Anti-Patterns

### Critical Anti-Patterns

**NEVER edit `internal/db/*.go`** — they are sqlc-generated. Edit `db/queries/*.sql` and regenerate.

**NEVER use CGO_ENABLED=1** — use modernc.org/sqlite instead.

**NEVER call repositories from handlers** — always use service layer.

**NEVER register routes outside `routes/routes.go`** — keep routing centralized.

**NEVER bypass auth middleware** for protected endpoints.

**NEVER use `$state` runes in .ts files** — only in .svelte files.

**NEVER hardcode API URLs** — use `PUBLIC_API_URL` environment variable.

### Request DTO Patterns

**CreateXRequest** (required fields):
```go
type CreateUserRequest struct {
    Username string `json:"username" validate:"required,min:3,max:255"`
    Email    string `json:"email" validate:"required,email"`
    Password string `json:"password" validate:"required,min:8"`
}
```

**UpdateXRequest** (pointer fields for partial updates):
```go
type UpdateUserRequest struct {
    Username *string `json:"username"`
    Email    *string `json:"email"`
    Password *string `json:"password"`
}
```

### Error Handling

Services return typed errors:
```go
func (s *YourService) YourMethod(ctx context.Context, id int64) error {
    // ...
    if notFound {
        return domain.ErrYourResourceNotFound
    }
    // ...
}
```

Handlers translate to HTTP codes:
```go
func (h *YourHandler) YourMethod(w http.ResponseWriter, r *http.Request) {
    // ...
    if errors.Is(err, domain.ErrYourResourceNotFound) {
        response.Error(w, http.StatusNotFound, "Resource not found")
        return
    }
    // ...
}
```

### Response Format

Always use JSON with snake_case fields and ISO 8601 timestamps:

```json
{
    "id": 123,
    "name": "Resource Name",
    "created_at": "2026-01-15T10:30:00Z",
    "updated_at": "2026-01-15T10:30:00Z"
}
```

### Testing Tips

- Use `t.Cleanup()` for resource cleanup
- Test both success and error cases
- Use `httptest.Server` for integration tests
- Follow the `testify/require` pattern for assertions

## Security Notes

- Set a strong admin password via `auth.initial_admin_password` — **required for production**
- Use `.env` files for secrets, never commit them
- SQLite uses WAL mode for better performance
- All functional testing must be done on the test server (your-test-server)
